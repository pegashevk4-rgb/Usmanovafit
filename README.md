# Усманова.Fit — тестовое задание (вайбкодинг)

Лендинг по мотивам страницы [usmanovafit.gymteam.ru/mainpage](https://usmanovafit.gymteam.ru/mainpage), собранный в формате вайбкодинга для тестового задания на позицию «Вайбкодер / Сборщик лендингов и воронок на AI (Усманова)».

## Что сделано

- Повторены три экрана лендинга:
  - **Hero**: первый экран с УСМАНОВА.FIT, фоном и оффером.
  - **О тренере**: блок с описанием Кати Усмановой и карточкой «580K+ участниц».
  - **Программы**: сетка карточек домашних тренировок (Метод Усмановой, марафоны и курсы).
- Реализован фиксированный **хедер** с логотипом и навигацией по якорям.
- Сделано **бургер‑меню** для мобильной версии с удобным закрытием:
  - по кнопке,
  - по клику на пункт меню,
  - по клику по фону.
- Добавлены плавные **анимации появления** секций через IntersectionObserver.
- Собрана демо‑**форма заявки**:
  - при отправке показывает сообщение «Заявка отправлена! Это демо-форма тестового задания.»
  - данные никуда не отправляются, форма используется как демонстрация работы.
- Реализован **адаптив** под планшеты и телефоны:
  - перестройка сеток,
  - корректные отступы,
  - читаемость блоков и карточек на малых экранах.
- Все изображения переведены в локальные файлы (`images/*.avif`) для быстрой загрузки без зависимости от VPN и внешних стоков.

## Технологии

- **Вайбкодинг / AI**:
  - Claude — генерация базовой структуры лендинга (HTML/CSS/JS).
  - Perplexity (GPT‑5.1) — помощь в архитектуре верстки, адаптиве, правках бургер‑меню, фиксированного хедера и подключении локальных изображений.
- **Ручная доработка**:
  - Редактор кода (VS Code).
  - Чистый HTML, CSS и Vanilla JS без фреймворков.
  - Git + GitHub для версионирования и деплоя через GitHub Pages.

## Структура проекта

- `index.html` — разметка лендинга (hero, «О тренере», «Программы», CTA, форма, футер).
- `style.css` — все стили, в том числе адаптивные `@media` и анимации.
- `script.js` — логика хедера, бургер‑меню и анимаций появления.
- `images/` — локальные изображения в формате AVIF.
